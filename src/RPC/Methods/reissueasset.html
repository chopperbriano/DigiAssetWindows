reissueasset

reissueasset

Issues more of an existing UNLOCKED asset. The wallet must control the address the asset was originally issued from, and that address must hold at least one DigiByte only UTXO (the assetId is derived from the first input, so the reissuance must spend from the issuer address - send a little DGB to it first if it holds no coins). The reissuance reuses the existing metadata and keeps the asset's current rules.

Input Parameters

asset (string or integer): assetId or assetIndex of the unlocked asset.

amount (number or string): Additional assets to create in display units.

options (object, optional): 

{
    "toAddress": "dgb1...."       // address or domain the new assets go to.
                                  // defaults to a new wallet address
}

Output Format

{
    "txid": "....",      // txid of the reissuance transaction
    "assetId": "Ua...."  // id of the asset (same as the original)
}

Examples

digiasset_core-cli reissueasset "Ua4zrzWkUwojQT9vkr2aB7snw7czocjxv48Rd2" 100

curl --user myusername --data-binary '{"jsonrpc": "1.0", "id":"curltest", "method": "reissueasset", "params": ["Ua4zrzWkUwojQT9vkr2aB7snw7czocjxv48Rd2", 100]}' -H 'content-type: text/plain;' http://127.0.0.1:14024/
