sendasset

Sends DigiAssets from the wallet to an address. The wallet automatically picks which asset UTXOs to spend, sends any asset change back to the wallet and pays the transaction fee in DigiByte. Requires DigiByte Core to be built with wallet support and the wallet to be unlocked.

Input Parameters

address (string): Recipient address or DigiByte domain.

asset (string or integer): assetId or assetIndex of the asset to send. Assets that can have more than 1 index (unlocked assets not starting with "L" or "Ua") must be referred to by assetIndex.

amount (number or string): Amount to send in display units. Sending "1.5" of an asset with 2 decimals sends 150 of the smallest unit.

options (object, optional): 

{
    "changeAddress": "dgb1....",  // address asset change should go to.
    "dryrun": false               // when true builds the transaction and returns its cost
                                  // (outputs, estimatedMinerFee, estimatedTotal) without
                                  // broadcasting anything
                                  // defaults to a new wallet change address
}

Output Format

txid (string) of the broadcast transaction

Examples

digiasset_core-cli sendasset "dgb1qtqt4vrsjfnncr7wjvdvhw7evgzsyj39kaxhg6z" "La5x1DEP8J2XiViw9J7bhCFXVsJnfLDBBWo6aS" 5
