burnasset

Permanently destroys DigiAssets held by the wallet. The burn is written on chain so total supply verifiably drops. Any asset change goes back to the wallet and the transaction fee is paid in DigiByte. Requires DigiByte Core to be built with wallet support and the wallet to be unlocked.

Input Parameters

asset (string or integer): assetId or assetIndex of the asset to burn. Assets that can have more than 1 index (unlocked assets not starting with "L" or "Ua") must be referred to by assetIndex.

amount (number or string): Amount to burn in display units. Burning "1.5" of an asset with 2 decimals burns 150 of the smallest unit.

options (object, optional): 

{
    "changeAddress": "dgb1....",  // address asset change should go to.
    "dryrun": false               // when true builds the transaction and returns its cost
                                  // (outputs, estimatedMinerFee, estimatedTotal) without
                                  // broadcasting anything
                                  // defaults to a new wallet change address
}

Output Format

txid (string) of the broadcast transaction

Examples

digiasset_core-cli burnasset "La5x1DEP8J2XiViw9J7bhCFXVsJnfLDBBWo6aS" 5
