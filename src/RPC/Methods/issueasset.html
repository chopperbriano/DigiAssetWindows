issueasset

issueasset

Creates (issues) a new DigiAsset. The asset metadata gets published to IPFS and the issuance transaction is funded, signed and broadcast by the wallet. Requires DigiByte Core to be built with wallet support, the wallet to be unlocked, and a running IPFS node.

Input Parameters

asset (object): Description of the asset to create. 

{
    "name": "My Asset",           // (string required) asset name
    "amount": 1000,               // (number or string required) number of assets to create
                                  //     in display units("1.5" with 2 decimals creates 150
                                  //     of the smallest unit)
    "decimals": 0,                // (integer 0-8 optional default 0) decimal places
    "locked": true,               // (bool optional default true) true means no more can
                                  //     ever be issued
    "aggregation": "aggregable",  // (string optional default "aggregable") one of
                                  //     "aggregable", "hybrid", "dispersed"
    "description": "about it",    // (string optional) stored in the metadata
    "urls": [{                    // (array optional) urls stored in the metadata
        "name": "icon",
        "url": "ipfs://...",
        "mimeType": "image/png"
    }],
    "userData": {},               // (object optional) any extra metadata
    "toAddress": "dgb1q....",     // (string optional) address or domain the new assets
                                  //     go to.  defaults to a new wallet address
    "metadata": {}                // (object optional advanced) full metadata document to
                                  //     publish as is.  when used name/description/urls/
                                  //     userData are ignored
}

Note: rule encoding (royalties, approval lists, voting, expiry, etc.) is not supported yet.

Output Format

{
    "txid": "....",      // txid of the issuance transaction
    "assetId": "La....", // id of the newly created asset
    "cid": "b...."       // IPFS cid of the published metadata
}

Examples

digiasset_core-cli issueasset '{"name":"My Coin","amount":1000,"decimals":2,"description":"Test asset"}'

curl --user myusername --data-binary '{"jsonrpc": "1.0", "id":"curltest", "method": "issueasset", "params": [{"name":"My Coin","amount":1000}]}' -H 'content-type: text/plain;' http://127.0.0.1:14024/
