getnodestats

getnodestats

Returns a JSON snapshot of this node's current operational state: build version, chain-analyzer sync height, local asset counts, and the bitswap and permanent-coverage figures cached by the dashboard's background pollers. Running it on two nodes lets you compare them side by side to verify they are processing the same chain and serving the same DigiAsset content. Missing subsystems report zero or a "not probed" state rather than erroring, and the response is never cached so it can be polled.

Arguments

None.

Result

An object snapshot of the node's state.

NameTypeDescription
buildVersionstringBuild version string, e.g. "0.3.0-win.31".
syncHeightnumericChain analyzer's current block height (0 if the analyzer is not set).
assetCountnumericDistinct assetIds in the local database. Undercounts when an assetId has been reissued; see topAssetIndex for the pagination-correct total.
topAssetIndexnumericHighest assetIndex assigned. This is what listlastassets / listassets are keyed on, so paginating explorers should use it as the total.
bitswapobjectBitswap serving stats: probed (bool), available (bool, IPFS API reachable on the last poll), blocksSent (numeric), dataSent (numeric bytes) and blocksPerMin (numeric, outbound rate from the last 30-second window).
permanentCoverageobjectPermanent-list coverage: checked (bool, whether the scan has completed at least once), tracked (numeric, assetIds the permanent list tracks), have (numeric, of those, how many are in the local assets table) and missingCount (numeric, tracked minus have).

Examples

DigiAssetWindows-cli.exe getnodestats

curl --user myusername --data-binary '{"jsonrpc": "1.0", "id": "curltest", "method": "getnodestats", "params": []}' -H 'content-type: text/plain;' http://127.0.0.1:14024/
