getassetdata

getassetdata assetIndex|assetId ( txid vout excludeIPFS )

Returns the full record of a single DigiAsset as JSON. The asset can be identified by its node-specific assetIndex, or by assetId. Assets whose assetId maps to more than one index (ids that do not start with L or Ua) also require a txid and vout of any transaction involving the asset to disambiguate.

Arguments

Two usages are supported. Usage 1 looks up by assetIndex; Usage 2 looks up by assetId.

#NameTypeRequiredDescription
1assetIndex / assetIdnumeric or stringrequiredA numeric assetIndex (Usage 1) or a string assetId (Usage 2).
2txidstringoptionalUsage 2 only. 64-character txid of any transaction involving the asset, used to select the correct index when an assetId has multiple indexes.
3voutnumericoptionalUsage 2 only. Output index within the given txid.
4excludeIPFSbooleanoptionalIf true, omits IPFS content from the result. Defaults to false. In Usage 1 (assetIndex) this is passed as argument 2 instead.

Result

A JSON object representing the DigiAsset (metadata, counts, PSP membership, and optional IPFS content). Note that DigiByte locked in asset UTXOs is included in the DigiByte total; the actual spendable amount will be less.

NameTypeDescription
(asset object)objectFull DigiAsset record including assetIndex, assetId, CID, counts, and issuance metadata.
