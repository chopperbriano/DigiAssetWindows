burnasset

burnasset asset amount ( options )

Permanently destroys DigiAssets held by the wallet. The burn is written on chain as a transfer instruction targeting the burn marker, so the total supply verifiably drops. The wallet selects the asset UTXOs to spend, returns any asset change (of this or other assets carried on the same UTXOs) to the wallet, and pays the miner fee in DigiByte. DigiByte (assetIndex 1) can not be burned.

Arguments

#NameTypeRequiredDescription
1assetstring or numericrequiredAsset to burn, given as an assetId (string) or assetIndex (integer). Assets with more than one index (unlocked assets whose id does not start with "Ua") must be identified by assetIndex.
2amountnumeric or stringrequiredAmount to burn in display units (an integer, decimal number, or numeric string).
3optionsobjectoptionalObject with optional members: "changeAddress" (string) the address any asset change should go to, defaulting to a new wallet change address; "dryrun" (bool, default false) build the transaction and return the cost breakdown WITHOUT broadcasting.

Result

By default the txid (string) of the broadcast transaction. When "dryrun" is true, an object describing the cost is returned instead of broadcasting.

NameTypeDescription
txidstringTransaction id of the broadcast burn (default response).
outputsstringDry run only. Total value of the transaction outputs in decimal DGB.
estimatedMinerFeestringDry run only. Estimated miner fee in decimal DGB.
estimatedTotalstringDry run only. Estimated total cost (outputs plus miner fee) in decimal DGB.
satsobjectDry run only. The same "outputs" and "estimatedMinerFee" figures expressed in sats.

Examples

DigiAssetWindows-cli.exe burnasset La5PefА7Rxs7mrvyv9nJ 100
