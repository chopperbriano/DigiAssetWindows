issueasset

issueasset {"name":...,"amount":...,...}

Creates (issues) a new DigiAsset. The asset's metadata is published to IPFS and the issuance transaction is funded, signed and broadcast by the wallet. By default the metadata is stored in the public permanent storage pool, which charges $1.20 USD per MB, paid in DGB as an extra output on the issuance. A single object argument describes the asset; only "name" and "amount" are required.

Arguments

Takes exactly one object argument. Its members are:

#NameTypeRequiredDescription
1(object)objectrequiredAn object describing the asset (see members below).
 namestringrequiredAsset name (must be non-empty).
 amountnumeric or stringrequiredNumber of assets to create in display units.
 decimalsnumericoptionalNumber of decimal places, 0 to 7. Defaults to 0.
 lockedbooleanoptionalTrue (default) means no more can ever be issued.
 aggregationstringoptionalOne of "aggregable" (default), "hybrid" or "dispersed".
 descriptionstringoptionalDescription stored in the metadata.
 urlsarrayoptionalArray of url objects ({"name":..,"url":..,"mimeType":..}) stored in the metadata.
 userDataobjectoptionalAny extra data to store in the metadata.
 toAddressstringoptionalAddress or domain the new assets should be sent to. Defaults to a new wallet address.
 metadataobjectoptionalAdvanced: a full metadata object to publish as is. When used, name/description/urls/userData are ignored.
 pspbool, numeric or arrayoptionalWhich permanent storage pool(s) store the metadata. true = the public pool (default), false = no pool, an integer = a single pool index (see getpsp), or an array of integers for several pools.
 rulesobjectoptionalRestrictions encoded with the asset. Supports changeable, approval, royalty, kyc, geofence, voting, expiry and deflation.
 dryrunbooleanoptionalIf true, build everything and return the cost breakdown WITHOUT broadcasting (the metadata is still published to the local IPFS node so storage can be priced). Defaults to false.

Result

By default an object identifying the newly created asset. When "dryrun" is true, a cost-breakdown object is returned instead.

NameTypeDescription
txidstringTransaction id of the issuance transaction.
assetIdstringId of the newly created asset (empty if it must be looked up once the tx confirms).
cidstringIPFS cid of the published metadata.
(dryrun)objectDry run only. Instead of the above, an object with cid, assetOutputs, ruleOutputs, pspFee, estimatedMinerFee, estimatedTotal (decimal DGB strings) and a sats object with the same figures in sats.

Examples

DigiAssetWindows-cli.exe issueasset "{\"name\":\"MyToken\",\"amount\":1000,\"decimals\":2}"

curl --user myusername --data-binary '{"jsonrpc": "1.0", "id": "curltest", "method": "issueasset", "params": [{"name": "MyToken", "amount": 1000, "decimals": 2}]}' -H 'content-type: text/plain;' http://127.0.0.1:14024/
