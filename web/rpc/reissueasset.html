reissueasset

reissueasset asset amount ( options )

Issues more of an existing UNLOCKED asset. The wallet must control the address the asset was originally issued from, and that address must hold at least one DigiByte-only UTXO, because the assetId is derived from the first input and the reissuance must spend from the issuer address (send a little DGB there first if it has no coins). The reissuance reuses the existing metadata and keeps the asset's existing rules.

Arguments

#NameTypeRequiredDescription
1assetstring or numericrequiredThe unlocked asset to reissue, given as an assetId (string) or assetIndex (integer).
2amountnumeric or stringrequiredAdditional assets to create in display units (an integer, decimal number, or numeric string).
3optionsobjectoptionalObject with optional members: "toAddress" (string) the address or domain the new assets go to, defaulting to a new wallet address; "dryrun" (bool, default false) build the transaction and return the cost breakdown WITHOUT broadcasting.

Result

By default an object identifying the reissuance. When "dryrun" is true, a cost-breakdown object is returned instead.

NameTypeDescription
txidstringTransaction id of the reissuance transaction.
assetIdstringId of the asset (the same as the original).
(dryrun)objectDry run only. Instead of the above, an object with outputs, estimatedMinerFee, estimatedTotal (decimal DGB strings) and a sats object with the same figures in sats.

Examples

DigiAssetWindows-cli.exe reissueasset 42 500

curl --user myusername --data-binary '{"jsonrpc": "1.0", "id": "curltest", "method": "reissueasset", "params": [42, 500]}' -H 'content-type: text/plain;' http://127.0.0.1:14024/
